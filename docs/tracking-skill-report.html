Tracking Agent Presentation

Tracking Agent

一个把“看得快”和“想得清”接在一起的目标跟踪方案：
前面用高频视觉链路持续看人，后面用 Tracking Agent 负责目标理解、恢复判断和记忆延续。

核心结论 跟踪难点不在看见人，而在持续确认“是不是同一个人”
方法特点 高频链路负责稳定，慢思考只在关键时刻介入
最终目标 让目标跟踪真正服务于智能跟随等机器人场景
00
01 / 核心问题

核心 message：目标跟踪真正难的不是检测，而是目标同一性

机器人真正要回答的，不是“这一帧里有谁”，而是“用户说的那个人，现在还是不是同一个人”。

真实难点

遮挡、交叉、转身、短时离场后，身份很容易断掉。

业务后果

一旦认错人，后续跟随、导航和交互都会建立在错误目标上。

为什么单靠视觉不够

视觉会告诉你“像不像”，但不会告诉你“用户最初说的是谁”。

为什么需要智能体

必须把自然语言、历史记忆和当前画面一起放进判断里。

这一页只讲一个结论

对真实机器人来说，目标跟踪不是一个“把框画准”的问题，而是一个“能不能持续跟住用户真正指定的那个人”的问题。

所以它天然需要三样东西一起工作：

  • 高频视觉链路，保证系统一直看得见世界。
  • 轻量身份证据，帮助判断是否还是同一个人。
  • Tracking Agent，负责在关键时刻做更高层的判断。
01
02 / 方法框架

核心 message：方法不是单点模型，而是一条“看见 → 保持 → 判断 → 记住”的完整链路

01

目标检测
YOLO

先把每一帧画面里的候选人找出来，为后续链路提供稳定输入。

产出 当前画面中的人框与候选列表
02

短时跟踪
ByteTrack

把相邻画面串起来，为每个候选人保持短时间内的连续身份。

产出 连续轨迹、候选编号、短时位置变化
03

轻量身份辅助
端侧 ReID

在端侧补充身份线索，帮助系统在相似人之间判断“像不像同一个人”。

产出 相似度证据、前后视角的身份辅助线索
04

Tracking Agent
高层判断

结合用户的话、当前画面和历史记忆，决定是继续跟、等待恢复还是重新确认。

产出 目标判断、恢复决策、异常处理策略
05

记忆更新
持续服务下一轮

把本轮确认过的图像证据和文字记忆留下来，服务后续判断。

产出 目标正面图、背面图、文字摘要、历史判断
这页只讲一个结论

这套方法的价值，不在于某一个模块更强，而在于它把“高频视觉稳定性”和“高层目标判断”接成了同一条链，因此既能跑得动，又能在关键时刻做出更像人的判断。

03 / 模型看到什么

核心 message：Tracking Agent 不只看当前画面，还会带着目标记忆一起判断

这里最大的不同是：Tracking Agent 不是只看一张当前图，而是把“用户最初说的话”“当前画面上的候选人”“目标之前长什么样”“最近几轮发生了什么”一起放进判断里。所以它更像是在“理解一个目标”，而不是在“做一次图片分类”。

输入一：当前工作画面
这里直接展示的是 viewer 的真实工作画面。可以清楚看到：在多人、易混淆的场景里，系统最终锁定的是前景中背对镜头的 `14` 号候选目标。
输入二：目标记忆
真实文字记忆摘录：
男性，戴眼镜，穿黑色圆领短袖T恤与卡其色运动短裤，脚穿白色运动鞋和白色中筒袜；正面可见胸前白色小标识。与相似人的关键区别是短裤颜色与长度，以及上衣前胸的小标识。

这组前后图来自同一条真实会话：左图是多人场景里第一次锁定目标时的背身视角，右图是后续继续跟踪后重新确认到的正面视角。
语言输入

用户最初怎么描述这个人，例如“穿黑衣服的人”“刚进门的那个人”。

视觉输入

当前画面中有哪些候选人，每个候选人在哪里、看起来像谁。

图像记忆

此前已经确认过的目标正面和背面证据，用于后续比对。

文字记忆

目标特征、最近位置变化、最近几轮判断结果和关键线索。

04 / 为什么有效与下一步

核心 message：这套方法的优势是“平时省、关键时刻准”，下一步要把它做得更稳、更能落到场景里。

为什么这条路是对的

这套方法把绝大多数简单时刻交给高频视觉链路，把真正容易出错的边界时刻交给 Tracking Agent。这样做的好处是：平时不浪费高成本推理，关键时刻又能把自然语言、视觉证据和历史记忆一起拉进来做判断。

当前还差什么

目前系统已经能在真实链路里发生目标重确认和短时恢复，但还存在两个不足：一是有时会保守等待太久，二是重新找回后还不够稳。这意味着方法方向是对的，但证据强度和后续维持还要继续加强。

下一步方法演进

继续增强目标恢复能力，把轻量身份特征、目标正反面视觉记忆和时序约束结合得更紧，让系统在遮挡、交叉和短时离场时更快、更稳地把人找回来。

下一步场景落地

继续把 Tracking Agent 与后续跟随、导航等执行链路打通，让它从“能判断跟谁”走向“能稳定支撑完整的智能跟随场景”。

对老板来说，可以把 Tracking Agent 理解为一个“会判断、会记住、会保守处理异常”的目标跟踪层。
它不是替代前面的视觉链路，而是把前面的“看得见”真正升级成后面的“跟得住”。
