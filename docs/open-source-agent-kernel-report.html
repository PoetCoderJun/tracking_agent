Embodied Agent Architecture Report

通用内核，技能外挂，状态单源。

这份报告总结当前仓库已经完成的重构方向：把项目从 tracking 驱动的 runtime， 收敛成一个通用的 embodied agent kernel。backend 负责 session、turn orchestration、 perception 接口与 skill hook；tracking、speech、web_search 都下沉成独立 skills。

6 / 6 E2E Cases

当前主结论：新增 skill 已经不需要继续往 backend 里塞业务逻辑。 skill 只要实现约定 hook，就可以挂到 agent runner、viewer、rewrite 调度链上。
Kernel

现在的核心结构

backend 已经更像一个小型内核，而不是 tracking runtime。核心对象只围绕 session、runner、perception 和 skill hooks 组织。

Session State

backend/agent/session.py 与 backend/agent/session_store.py 负责统一的会话状态访问。

Turn Orchestration

backend/agent/runner.py 负责接收文本 turn、决定 direct skill path 或 Pi path、并应用结果。

Pi Protocol

backend/agent/pi_protocol.py 承担 prompt、payload 解析、subprocess 协议，不再混在 runner 里。

Skill Hooks

backend/skills.py 统一 discovery、route summary、turn context、direct init/turn、rewrite、viewer 模块聚合。

Flow

一次文本输入如何流转

1

文本输入进入 backend/cli.py，转给 PiAgentRunner。

2

runner 从 AgentSessionStore 读取当前 session，同时读取 perception snapshot。

3

backend 生成通用 route context，并按已启用 skills 生成 skill_context_paths。

4

如果某个 skill 声明它能直接处理当前 turn，就走 direct path；否则交给 Pi 选择 skill。

5

skill 返回统一 payload，runner 把结果写回 session，并可触发 skill-owned rewrite / IO。

Plugability

Skill Contract

目前 skill 可以按需实现这些 hook；不是每个 skill 都必须全部实现。

Hook 用途
build_route_summary 给 route context 提供 skill 自己的概览摘要
build_turn_context 给当前 turn 生成 skill 专属上下文文件
should_direct_init 允许 skill 在某些场景下绕过 Pi，直接走 deterministic path
process_direct_init 处理 direct init 型 turn
process_direct_turn 处理 direct runtime / continuation 型 turn
schedule_rewrite 异步或延迟型 skill-side IO / rewrite 任务
build_viewer_module 向 generic agent viewer 注入模块面板
Skills

当前验证过的技能

tracking speech web_search

tracking

runtime、viewer、rewrite worker 全部下沉到 skills/tracking/，backend 不再写 tracking 业务细节。

speech

dry-run CLI 与 skill discovery 已验证，证明非 tracking skill 也能自然挂在同一套 skill surface 上。

web_search

新增互联网搜索 skill，只动了 skills/web_search/ 和测试，没有再修改 backend 主干。

Viewer

通用 Agent Viewer

viewer 已经不再是 tracking-only。backend 只提供统一壳体，skills 自己提供模块面板。

{ "kind": "agent_viewer_state", "agent": { "...": "latest_result / conversation / turn_history" }, "observation": { "...": "latest_frame / stream_status / detection_count" }, "modules": { "tracking": { "...": "tracking-owned panel payload" } } }
Answer

是否已经自然即插即用？

结论是“基本成立”。新增 web_search skill 已经证明： backend 不需要因为新 skill 再增加专属业务逻辑。当前 skill 接入主要发生在 skills/<name>/ 自己和对应测试里。

  • 文本 turn 进入同一条 runner 主链
  • skill 能提供自己的 route summary / turn context / viewer module
  • tracking 不再定义 backend 的形状
