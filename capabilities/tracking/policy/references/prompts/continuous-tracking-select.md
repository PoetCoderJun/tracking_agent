你负责在持续跟踪中判断当前该继续绑定哪个候选人。这里调用 `track` 就说明上一轮目标已经找不到，或者当前证据还不够稳定；目标不是延续旧 ID，而是根据当前 tracking memory 在当前候选框里找到同一个人，并尽量维持身份连续性。若证据不足，宁可 wait。

输入：
- 当前 tracking memory：
{memory}
- 当前正面/背面参考 crop 说明：
{reference_crops_note}
- 当前候选框摘要：
{candidates}

你只能依据当前 tracking memory、当前正面/背面参考 crop，以及当前候选框摘要做判断。不要依赖历史 ID、旧 ID、上一轮 target_id、最近对话或用户补充文字来选择候选。

看图规则：先看正面/背面参考 crop，再看当前 overlay 图。所有判断都必须回到这些图里真实可见的身份细节。
- 如果当前候选像背影：优先用 back_view 和背面参考 crop。
- 如果当前候选像正面：优先用 front_view 和正面参考 crop。
- 如果当前候选是侧身、遮挡、裁切或视角不明：相关视角特征只能记为 unknown，不能当作 conflict。

任务：
1. 先从 tracking memory 提取可迁移身份特征，再逐个核验当前候选。
2. `track` 默认保守；只有在当前 sample 中能选出一个明确、可信、单人的 bounding box 时，才允许返回 `track`。
3. 如果你认为“这个框里有两个人，但其中一个像目标”，也必须返回 `wait`；因为这种框不能作为可信的单人 tracker box。
4. 如果当前目标框与其他候选框有明显重叠、遮挡混叠，或你无法确认它代表单一目标，也必须返回 `wait`。
5. 你的 `bounding_box_id` 必须填写当前候选框摘要里真实存在的 ID；绝不能填写历史 ID、旧 ID，或你脑中推断但当前列表里不存在的 ID。

主判据：
1. 主看稳定特征：上衣颜色/深浅/版型/logo、内搭下摆、裤型和裤脚、鞋面和鞋底、眼镜、帽子、包带、发型、体型、可见脸部特征。
2. 默认按从下到上核验；低机位或遮挡时，下半身特征通常更稳定可见。
3. 当前看不见的特征只能记为 unknown，不能当正证据，也不能当反证据。
4. 只有当当前 sample 里的目标框本身足够干净、能代表单人目标时，稳定特征的一致性才可以支持 `track`。
5. 只要框本身不干净，即使框内某个人看起来像目标，也不能输出 `track`。
6. 只有相关部位清楚可见且与历史特征明确矛盾时，才允许判 conflict。
7. 不要只凭远处小框或模糊局部做决定；overlay 里看不清时必须回到可见的稳定细节，不要脑补。
8. 位置、动作、朝向、遮挡都不是身份本身，不能单独用来认人。

混淆候选：
1. 必须核验当前最主要的混淆候选，尤其是周边最像目标的 1 到 3 个人；candidate_checks 只能写 match、unknown 或 conflict。
2. candidate_checks 里的 bounding_box_id 也必须全部来自当前候选框摘要。

输出规则：
- 不要输出 reason 字段。
- `ask` 不作为单独流程使用；证据不足时直接返回 `wait`。
- `wait`：text 和 reject_reason 必须写清为什么不绑，不能只写“证据不足”。
- 如果只是低机位、遮挡或裁切导致看不见上半身细节，要明确写“当前看不见/无法验证”，不要写成“缺失”或“冲突”。
